# Peer Review

## 适用场景

用户提供论文草稿、章节或投稿目标，希望得到类似同行评审的初审意见。

## 输入格式

- Markdown、TXT 或 LaTeX 文件路径。
- 论文草稿文本。
- 可选：目标会议或期刊。

## 输出格式

- 总体评价。
- 分章节评价。
- 创新性、技术正确性、实验充分性、写作质量评价。
- Major Weaknesses 和 Minor Weaknesses。
- 审稿人可能提问。
- 模拟审稿结论。
- 修改建议清单。

## 注意事项

- Markdown 和 LaTeX 草稿路径应优先调用 `research_review_draft`。
- 不要把长草稿全文一次性塞入上下文；应使用工具返回的章节分块评审结果。
- LaTeX 草稿会静态展开常见 `input/include` 子文件，识别简单宏、公式、图表、表格、附录和引用关系；无法解析的子文件或宏应作为 warning 写入评审意见。
- 如果 Zotero 已配置，`research_review_draft` 会检索用户文献库并输出引用完整性提示；该检查只能提示可能缺失的相关工作，不能替代人工核对参考文献。
- Zotero 未配置时不应影响 `/review` 主流程。
- 不要直接覆盖、重写或修改原论文草稿，只保存评审报告。
- 保持严谨，不要为了鼓励用户而降低评审标准。
- 如果草稿缺少章节，直接指出缺失部分。
- 生成最终 Markdown 后使用 `research_write_report` 保存到 `reviews`。

## 质量检查清单

- 是否覆盖主要章节。
- 是否支持 Markdown 和 LaTeX 输入。
- 是否说明长文本已经按章节分块审查。
- 是否检查公式 label、图表 caption、引用关系和附录内容。
- 是否包含 Zotero 引用完整性检查或未配置提示。
- 是否区分 major 和 minor 问题。
- 是否给出模拟接收/拒稿结论。
- 是否没有覆盖原论文。
- 是否保存最终报告。
