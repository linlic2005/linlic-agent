# Experiment Critic

## 适用场景

用户提供实验计划、实验章节草稿或 baseline 方案，希望评估实验设计是否能支撑论文贡献。

## 输入格式

- 自然语言实验方案。
- Markdown/TXT 文件路径。
- 可选：目标会议、数据集、baseline、指标、消融实验。

## 输出格式

- 实验目标清晰度。
- 假设可检验性。
- 数据集、baseline、指标和消融实验评价。
- 数据泄漏和不公平比较风险。
- 统计显著性和可复现性建议。
- 必须补充和可选增强实验。
- 最终评分。

## 注意事项

- 对实验不足保持直接，但建议必须可执行。
- 不要只给笼统评价，要指出缺失的 baseline、指标或消融位置。
- 生成最终 Markdown 后使用 `research_write_report` 保存到 `reviews`。

## 质量检查清单

- 是否检查数据泄漏风险。
- 是否检查 baseline 是否充分。
- 是否给出必须补充实验。
- 是否保存最终报告。
