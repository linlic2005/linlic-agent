# Novelty Check

## 适用场景

用户提出研究想法，希望判断是否可能已有类似工作、创新性风险在哪里、技术和实验是否可行。

## 输入格式

- 自然语言研究想法。
- 可选：目标领域、目标会议、已有方法、计划数据集、实验资源。

## 输出格式

- 用户想法复述。
- 想法拆解。
- 检索关键词和英文检索式。
- 相似工作风险类型。
- 创新性风险、技术可行性、数据可得性、实验成本。
- 修改后的研究定位。

## 注意事项

- novelty check 不是正式查重，不能替代 iThenticate、Turnitin 或学校查重系统。
- 当前 MVP 通过 `research_check_idea` 先尝试检索用户 Zotero 文献库，再复用 `paper-search`，按 Semantic Scholar、OpenAlex、arXiv 顺序检索相似工作。
- Zotero 未配置或 Web API 请求失败时不能中断 `/idea`，只记录提示并继续外部数据库检索。
- 相似工作判断基于标题、摘要和关键词重合度，不能替代人工精读和系统性 related work 调研。
- 生成最终 Markdown 后使用 `research_write_report` 保存到 `reports`。

## 质量检查清单

- 是否明确查重局限。
- 是否拆解研究对象、场景、方法、贡献。
- 是否优先检查用户 Zotero 文献库，或明确说明 Zotero 未配置。
- 是否列出最相似相关工作，并说明相似点、不同点和创新性威胁程度。
- 是否指出高风险相似工作类型。
- 是否覆盖技术可行性、数据可得性和实验成本。
- 是否保存最终报告。
